🚀 GOWTHAM CAREERPILOT AI - DETAILED SYSTEM SPECIFICATION & IMPLEMENTATION PLAN
================================================================================

This document details the overall architecture, frontend views, backend systems, 
database routing interfaces, AI strategy, and deployment configuration for 
Gowtham CareerPilot AI.

--------------------------------------------------------------------------------
1. PROJECT VISION & OBJECTIVE
--------------------------------------------------------------------------------
Gowtham CareerPilot AI is a production-ready, full-stack platform designed to
help job seekers navigate the modern employment landscape. By combining advanced
generative AI (Google Gemini) with clean management workflows, the app automates
resume ATS audits, maps out personalized learning paths, forecasts callback rates,
and replaces scattered job-hunting spreadsheets with a unified analytics dashboard.

--------------------------------------------------------------------------------
2. TECHNICAL STACK OVERVIEW
--------------------------------------------------------------------------------
• Frontend client:
  - React 19 (Main component library)
  - TypeScript (Strong typing for layouts and API responses)
  - Tailwind CSS v4 (Sleek, responsive utility styles)
  - Motion / Framer Motion (Fluid micro-animations and page transitions)
  - Lucide React (Sleek vector icon sets)
  - Recharts (Interactive SVG charting engine for analytics)

• Backend server:
  - Node.js & Express (RESTful API endpoints)
  - TSX (Direct runtime execution of TypeScript modules)
  - Esbuild (Server-side compilation into single CommonJS file)

• Data Layer:
  - Repository Provider Design Pattern (Decoupled data access interface)
  - JSON File Database (Zero-config local file-system database `db.json`)
  - MySQL Database (Production relational storage server)

• Artificial Intelligence:
  - Google Gemini 3.5 Flash & 1.5 Flash models
  - `@google/genai` (Official modern Google Gen AI Node.js SDK)
  - JSON Structured Output Schemas (Forced API response structures)

• Authentication:
  - JSON Web Tokens (JWT) for stateless session validation
  - SHA256 hashed password verification

4. DETAILED FEATURE SPECIFICATION
--------------------------------------------------------------------------------

4.1 STATS & ANALYTICS DASHBOARD Overview
  - Application Funnel: Summarizes Applied, Interview, Offer, and Rejections.
  - To-Do Reminders: Scrapes tracking pipeline to list upcoming scheduled interviews.
  - Performance Widgets: Displays progress percentages (e.g. Success Callback Rate).
  - Visualization: Recharts graphs mapping monthly trends and top applied domains.

4.2 APPLICATIONS TRACKER
  - Full CRUD operations: Create, read, update, delete job applications.
  - Pipeline Fields: Company, Role, Status, applied date, location, expected salary,
    job description, and text follow-up notes.
  - UI Grid: Clean sorting and search filters to locate specific listings instantly.

4.3 SAAS RESUME STUDIO & ATS AUDITOR
  - AI PDF Parser (OCR): Reads raw PDF uploads, encodes files in base64, and extracts
    plain text directly using the `gemini-1.5-flash` model.
  - ATS Matcher: Scores compatibility from 0 to 100% by comparing resume text against
    job descriptions. Lists key missing skills and recommends targeted changes.
  - Action-Verb Bullet Optimizer: Takes weak resume sentences and rewrites them into
    STAR framework points (Situation, Task, Action, Result) with quantitative metrics.
  - Resume from Scratch: Interactively generates structured resume JSON models.

4.4 CAREER GPS (Roadmap Generator)
  - Custom Milestones: Synthesizes a 4-step roadmap to transition from a list of
    current skills to a target job role.
  - Step details: Domain title, description, estimated timeline, online courses, 
    and official certifications.
  - Database caching: Users can generate multiple roadmaps and review them.

4.5 COPILOT AI FORECASTER
  - Interview Probability: Forecasts an upcoming callback rate based on tracker
    status history and active resume skills.
  - Market demands: Identifies hiring trends and average salary figures.
  - Advice: Generates customized coaching recommendations.

5. DETAILED DATA ARCHITECTURE & SWAPPING SCHEMES
--------------------------------------------------------------------------------
The project isolates database operations behind clean repository interfaces.
The variable `process.env.DATABASE_PROVIDER` decides which provider is loaded at runtime:

                          ┌─────────────────────────┐
                          │   RepositoryProvider    │
                          └────────────┬────────────┘
                                       │
                ┌──────────────────────┴──────────────────────┐
    DATABASE_PROVIDER="json"                      DATABASE_PROVIDER="mysql"
                ▼                                             ▼
  ┌───────────────────────────┐                 ┌───────────────────────────┐
  │   JsonUserRepository      │                 │   MysqlUserRepository     │
  │   JsonJobRepository       │                 │   MysqlJobRepository      │
  │   JsonResumeRepository    │                 │   MysqlResumeRepository   │
  │   JsonAIAnalysisRepository│                 │   MysqlAIAnalysisRepos... │
  │   JsonRoadmapRepository   │                 │   MysqlRoadmapRepository  │
  └───────────────────────────┘                 └───────────────────────────┘
                │                                             │
                ▼                                             ▼
          [ db.json File ]                              [ MySQL DB ]

• JSON Database Provider:
  - Local setup: Writes are written directly to `db.json` in the root folder.
  - Serverless environment: When deployed on Vercel, the filesystem is read-only.
    To bypass EROFS errors, writes are automatically redirected to Vercel's writable
    `/tmp/db.json` buffer, which is auto-seeded from the initial template file.

• MySQL Database Provider:
  - Local/Production: Connects to a standard SQL database pool.
  - Migrations: `npm run migrate` runs `/scripts/migrate-db.ts`. It reads the current
    local `db.json` dataset, creates necessary relational tables, and inserts all
    user accounts, job records, and generated roadmaps into MySQL.

6. API ENDPOINTS SCHEMA DOCUMENTATION
--------------------------------------------------------------------------------

All endpoints (except login/register) expect a `Bearer <JWT_TOKEN>` header.

6.1 Authentication Routes:
  - POST `/api/auth/register` : Create user account.
    - Body: `{ email, password, name }`
    - Response: `{ token, user: { id, name, email, createdAt } }`
  - POST `/api/auth/login` : Authenticate user.
    - Body: `{ email, password }`
    - Response: `{ token, user: { id, name, email } }`
  - GET `/api/auth/me` : Validate JWT token and retrieve profile.
    - Response: `{ id, name, email, createdAt }`

6.2 Job Application Routes:
  - GET `/api/applications` : Retrieve all tracking logs for authenticated user.
    - Response: `JobApplication[]`
  - POST `/api/applications` : Log new application.
    - Body: `{ company, role, appliedDate, status, salary?, location?, jobDescription?, notes? }`
    - Response: `JobApplication`
  - PUT `/api/applications/:id` : Update application fields.
    - Body: `{ ...fields }`
    - Response: `JobApplication`
  - DELETE `/api/applications/:id` : Remove application.
    - Response: `{ message: "Successfully deleted application track." }`

6.3 Resume & AI Auditor Routes:
  - GET `/api/resumes` : Fetch uploaded resume drafts.
    - Response: `Resume[]`
  - POST `/api/resumes` : Upload custom parsed resume text.
    - Body: `{ filename, parseText, fileSize }`
    - Response: `Resume`
  - DELETE `/api/resumes/:id` : Delete draft files.
    - Response: `{ message: "Deleted resume version successfully." }`
  - POST `/api/resumes/parse-pdf` : Parse PDF file using Gemini OCR.
    - Body: `{ filename, base64Data }`
    - Response: `{ parseText: string }`
  - POST `/api/resumes/enhance` : Optimize resume text keywords via AI.
    - Body: `{ resumeText }`
    - Response: `{ enhancedText: string }`
  - POST `/api/resumes/fix-bullets` : Optimize bullet points using STAR format.
    - Body: `{ bulletPoints: string[] }`
    - Response: `{ bulletPoints: string[] }`
  - POST `/api/resumes/match-jd` : Match resume against job description.
    - Body: `{ resumeText, jobDescription }`
    - Response: `{ matchScore: number, missingKeywords: string[], recommendedChanges: string[] }`
  - POST `/api/resumes/optimize` : Align resume JSON structure to target role.
    - Body: `{ resumeData, targetRole }`
    - Response: `OptimizedResumeJSON`
  - POST `/api/resumes/generate-scratch` : Interactively generate JSON resume models.
    - Body: `{ name, targetRole, skills: string[], rawExperience }`
    - Response: `GeneratedResumeJSON`
  - POST `/api/resumes/:id/analyze` : Perform comprehensive ATS resume analysis.
    - Response: `AIAnalysis`
  - GET `/api/resumes/:id/analysis` : Retrieve latest ATS scorecard report.
    - Response: `AIAnalysis`

6.4 Career GPS & Forecasting Routes:
  - GET `/api/career/roadmaps` : Retrieve saved curriculum paths.
    - Response: `LearningRoadmap[]`
  - POST `/api/career/roadmap` : Construct new roadmap using AI.
    - Body: `{ currentSkills: string[], targetRole }`
    - Response: `LearningRoadmap`
  - GET `/api/analytics/dashboard` : Retrieve metrics compiled for graphics charts.
    - Response: `{ totalCount, statusCounts, monthlyTrends, domainDistribution, interviewSuccessRate, rejectionRatio, offerCount }`
  - GET `/api/analytics/insights` : Compile predictive callback/interview forecasts.
    - Response: `{ interviewProbability, probJustification, suggestedCompanies, missingSkillsSuggestions, overallAdvice }`

7. AI ORCHESTRATION & DOCKING SCHEMAS
--------------------------------------------------------------------------------
The server configures Gemini prompts utilizing `responseSchema` options to
enforce strict compliance with TypeScript interfaces. Here are key schemas used:

• Resume Audit Report Schema:
  - model: `gemini-3.5-flash`
  - systemInstruction: "Assess resume, score ATS compatibility out of 100, suggest keywords, rewrite bullets."
  - format: JSON Object `{ atsScore: number, missingKeywords: string[], suggestions: string[], improvedBulletPoints: string[] }`

• Career Roadmap Schema:
  - model: `gemini-3.5-flash`
  - systemInstruction: "Synthesize linear, 4-step actionable learning roadmaps with exact timelines and online courses."
  - format: JSON Array `[{ title: string, description: string, timeline: string, resources: string[] }]`

• Predictive Career Insights Schema:
  - model: `gemini-3.5-flash`
  - systemInstruction: "Predict callback rate, justify probability, recommend companies, suggest missing skills."
  - format: JSON Object `{ interviewProbability: number, probJustification: string, suggestedCompanies: [...], missingSkillsSuggestions: [...], overallAdvice: string }`

8. VERCEL SERVERLESS CLOUD DEPLOYMENT
--------------------------------------------------------------------------------
When deployed on Vercel, the platform uses serverless routes matching `vercel.json` configurations:
• Serverless Routing:
  - The entire backend Express app is compiled into serverless execution modules.
  - Rewrites target `server.ts` for all `/api/*` endpoints.
  - Static front-end assets are built with Vite into `dist/` and served statically.
• Ephemeral Storage:
  - A startup validation checker checks if `process.env.VERCEL === "1"`.
  - If true, writes are stored in the `/tmp/db.json` buffer space instead of the
    main file-system which is read-only.
